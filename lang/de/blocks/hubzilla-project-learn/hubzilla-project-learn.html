Dezentrales Netzwerk aus Hubs: ein Grid

Hubs sind Server, die miteinander kommunizieren durch die Verbreitung von Informationen im Netzwerk, oder "Grid". Es gibt keine zentrale Hubs oder einzelne Ausfallknoten. Einzlne Hubs im Netzwerk können je nach Wunsch mit vielen anderen oder mit einigen wenigen Hubs sich verbinden.

Kanäle und Verbindungen

Ein Kanal ist eine grundlegende Identität im Grid. Ein Kanal kann repräsentieren eine Vielzahl von unterschiedlichen Dingen: eine Person, ein Blog, oder ein Forum, um einige Anwendungsbeispiele zu nenen. Kanäle können miteinander Beziehungen aufbauen, um Informationen zu teilen, wobei die Zugriffsberechtigungen detailliert definiert sind. Die Kanaladresse hat ein vertrautes Format channel@hub.domain.

Nomadische Identität and Klonen

Nomadische Identität means true ownership of online identity. Mit Hubzilla bekommen Sie eine Identität, die Sie überall im Netzwerk mitnehmen können und kein Login auf einem Server. Sie können Ihr Kanal auf mehreren Hubs klonen, um die Ausfallsicherheit zu verbessern oder Zensur zu verhindern oder Sie können mit Ihrem Kanal von einem auf einen anderen Kanal komplett umziehen und Ihre Daten und Beziehungen/ Verbindungen mitnehmen.

Webbasierte Zugriffsberechtigungen

Eine typische Website ist isoliert und hat keinen Schimmer, wer einen Zugriff auf die Inhalte dieser Webseite hat. Die Zugriffskontrolle beschränkt sich dabei auf die Gewährung von Berechtigungen für einzelne Accounts auf der Website. Sie haben Pech, weenn Sie außerhalb der Website den Zugriff auf die Inhalte der Website beschränken wollen. Hubzilla erzwingt fein granulierte Berechtigungen für Informationen, die im Grid geteilt werden und Websites und Hubs sind Identity-sensitiv. Hubzilla enforces fine-grained permissions for information shared across the grid, and websites on hubs are identity-aware. Einmalige Single sign-on Anmeldung ermöglicht eine nahtlose Authenifikation überall auf unabhängigen Websites in einer Art und Weise, die zuvor nicht möglich war.

Got Zot?

Zot ist ein JSON-basiertes Framework für die Implementierung von dezentralen sicheren Kommunication und von dezentralen Diensten. Diese ist ein Alleinstellungsmerkmal, das unterscheidet Zot von anderen Protokollen bei der Realisierung der Kommunikation mit einer dezentralen Identität und Authentifikation Framework. Die Authentifikationskomponente ist vergleichbar mit dem OpenID-Konzept ist aber isoliert von DNS-basierten Identitäten. Wo dies möglich ist, geschieht die entfernte Authentifikation still und unsichtbar. Dies bietet einen Mechanismus für skalierbare webbasierte verteilte unauffällige Zugriffskontrolle.

  • komplett dezentrale Kommunikation
  • Isolation von der DNS-basierten Identität
  • ein mobiler Knoten
  • unsichtbare oder reduzierte "InteraKtion" bei einer entfernten Authentifikation
  • Leistungsstärke
