Децентральная сеть из хабов: грид

Хабы это серверы, которые обмениваются данными в сети "grid". Все хабы равноправны. Хабы могут быть в зависимости от потребности связаны с некоторыми или с многими другими хабами.

Каналы и контакты

Канал - это основной элемент в грид. Канал может представлять множество различных явлений: для примера человека, блог или форум. Каналы могут связываться с другими каналами для обмена информации в соответствии с прецизионными правами доступа. Каналы адресуются адресами в првычном формате channel@hub.domain.

Кочующий двойник и клонирование

Кочующий двойник означает настоящий контроль над онлайн-двойником. В Хабзилла ты заводишь двойника в грид, а не на определённом сервере и этот двойник может перемещаться по всему грид. Ты можешь завести две или больше копии двойника, чтобы улучшить устойчивость против цензуры и доступность, ты можешь переместиться с одного хаба на другой вместе с данными и с контактами.

Контроль доступа во всей сети

Типичный сайт изолирован и не имеет понятия, кто посещает его и контроль доступа ограничивается контролем для зарегистрированных на определённом сайте пользователей. Если вы хотите предоставить доступ к информации только определённым пользователям, не зарегистрированным на сайте, то вам не повезло. Хабзилла же предоставляет прецизионный доступ к информации, распространяемой в сети и сайты на хабах могут распознавать посетителей. Благодаря Single sign-on возможна простая аутентификация на независимых сайтах, что раньше не было возможно.

Изволите Zot ?

Zot - это базирующийся на JSON набор функций для реализации децентральной коммуникации и децентральных сервисов. Отличительные свойства Zot в сравнении с многими другими коммуникационными протоколами - это кочующий двойник и набор функций для аутентификации. Аутентификационная составляющая Zot подобна OpenID, но изолирована от идентификаторов на основе DNS. Причём возможна тихая и невидимая удалённая аутентификация. Таким образом реализуется механизм для распределённого доступа через интернет.

  • полностью децентральная связь
  • изоляция от DNS базированных идентификаторов
  • узел мобильности
  • невидимый или компактный обмен при удалённой аутентификации
  • быстродействие

Плугин

Hubzilla можно развивать дальше. Программируй вебапс как плугин и дополни возможности грид для передачи данных и управления доступом.

Набор инструментов

Язык Comanche для описания интернет-страниц для программирования модульных, реагирующих на конкретного пользователя элементов страниц. Эти модули могут быть использованы другими hubzilla-полбзователями.

Социальная сеть

Свойство социальности записано в Хабзилла ДНК, в её ядре. Используйте каналы для того, чтобы представить себя и связывайтесь с другими участниками Хабзилла грид. Возможен обмен сообщениями с социальными сетями Диаспора, Фриндика и др. при помощи плугинов.

Облачное хранилище файлов

Соедините облачное хранилище файлов вашего канала с вшим жестким диском, пользуясь WebDAV стандартом. Распространяйте или публикуйте ваши файлы при помощи вездесущной прецизионной системы управления доступом.

Внешний вид

Внешний вид может быть гибко изменён с встроенной поддержкой для изменения внешнего вида. Сделайте ваш Хаб неповторимым и подходящим для вашего сообщества или для вашей фирмы.

События

Координируйте и распространяйте календарные события c встроенной поддержкой функции календаря. Импорт и экспорт с поддержкой распространённых форматов.

API

Всё большее количество API функций позволяет программировать независимые аппс для использования Хабов, расширение выбора сетевых сервисов в грид на границе вашей фантазии.
